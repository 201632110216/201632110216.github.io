HTML Link





HTML uses hyperlinks to connect to another document on the network. You can find links in almost all web pages. Click on the link to jump from one page to another.

For an example:
How to create a hyperlink in the document?

<'html>
<'body>
<'p>
<'a href="/index.html">TEXT<'/a> It's a link to a page in this site.<'/p>
<'p><'a href="http://www.microsoft.com/">TEXT<'/a> It's a link to a web
page on the world wide web.<'/p>
<'/body>
<'/html>



HTML Hyperlink (link)

Hyperlinks can be a word, a word, or a set of words, or an image, and you can click on them to jump to a new document or a part of the current document.
When you move the mouse pointer to a link in the page, the arrow turns into a small hand.
We create links in HTML by using the <'a> tag.
There are two ways to use <'a> tags:
Create a link to another document by using the href property
Create a bookmark in a document by using the name attribute


HTML Link Syntax

The linked HTML code is simple. It's like this:

<'a href="url">Link text

The href property specifies the goal of the link.
The text between the start tag and the end tag is displayed as a hyperlink.
For an example:

<'a href="http://www.baidu.com/">Visit BaiDu<'/a>

The line above is shown as: Visit BaiDu.
Clicking on this hyperlink will take the user to the front page of BaiDu.
Hint: "link text" does not necessarily have to be text. Pictures or other HTML elements can be links.


HTML link - target property

Using the Target property, you can define where the linked document is displayed.
The line below opens the document in a new window:
<'a href="http://www.BaiDu.com/" target="_blank">Visit BaiDu


HTML link - Name property

The name attribute specifies the name of the anchor (anchor).
You can use the name property to create bookmarks in HTML pages.
Bookmarks are not displayed in any special way, which is invisible to the reader.
When using the named anchor (named anchors), we can create a direct jump to the named anchor (such as a section of the page) link, so users do not need to constantly scroll the page to find the information they need.
The grammar of named anchors:
<'a name="label">anchor(Text displayed on the page.)<'/a>

Hint: anchor's name can be anything you like.

Hint: you can use the ID property instead of the name attribute, and the named anchor is equally valid.


Basic considerations - useful tips:

Note: always add forward slashes to subfolders. If you write links like this, "href=" "http://www.w3school.com.cn/html", you will generate two HTTP requests to the server. This is because the server adds a forward slash to this address, and then creates a new request, like this: href= "http://www.w3school.com.cn/html/"".

Hint: named anchors are often used to create directories at the beginning of large documents. You can assign a named anchor for each chapter, and then link the links to these anchors to the top of the document. If you visit Baidu encyclopedia frequently, you'll find that almost every entry uses such navigation.

Hint: if the browser cannot find the named anchor that has been defined, it will be positioned at the top of the document. There will be no mistakes.
