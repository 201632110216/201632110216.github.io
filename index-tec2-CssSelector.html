CSS Selector





There are many selectors in CSS,but Little Beetle just teach three Frequently-used selector.There are : id selector,class selector,and element selector.


id Selector

The id selector can specify a particular style for the HTML element with a specific id.
The id selector to "#" to define.

The next two id selectors, the first one to define the element's color is red, and the second defined element is green:
#red {color:red;}
#green {color:green;}

In the following HTML code, the ID attribute is shown as red by the P element of red, and the P element of the ID attribute green is shown to be green.
<'p id= "red" > this paragraph is red.
<'p id= "green" > this paragraph is green.

Note: the ID property can only appear once in each HTML document. If you want to know why, please refer to the XHTML: website refactoring.


id Selectors And Derived Selectors

In modern layouts, ID selectors are often used to create derived selectors.
#sidebar p {
font-style: italic;
text-align: right;
margin-top: 0.5em;
}

The style above applies only to paragraphs that appear in elements of ID sidebar. This element is likely to be a div or a table element, although it can also be a table or other block level element. It can even be an inline element, such as <'em> or <'span>, but this usage is illegal, because it can not embed <'p> in inline elements in <'span>.


A selector, multiple uses

Even if the element marked as sidebar can only appear once in the document, the ID selector can be used many times as a derivative selector:
#sidebar p {
font-style: italic;
text-align: right;
margin-top: 0.5em;
}
#sidebar h2 {
font-size: 1em;
font-weight: normal;
font-style: italic;
margin: 0;
line-height: 1.5;
text-align: right;
}

Here, unlike the other P elements in the page, the P elements in sidebar are treated specifically, and at the same time, unlike all other H2 elements in the page, the H2 elements in sidebar are treated differently.


Separate Selector

id selector can function independently even if it is not used to create a derivative selector:
#sidebar {
border: 1px dotted #000;
padding: 10px;
}

According to this rule, the element id of sidebar will have a pixel wide black dot border, and there will be 10 pixels wide margins (padding, inner blank) around it. The old version of the Windows/IE browser might ignore this rule unless you specifically define the element that the selector belongs to:
div#sidebar {
border: 1px dotted #000;
padding: 10px;
}



Class Selector


In CSS, classes displayed by a dot:
.center {text-align: center}
n the example above, all HTML elements with center classes are centered.
In the following HTML code, both H1 and P elements have center classes. This means that both will follow the rules in the ".Center" selector.
<'h1 class="center">
This heading will be center-aligned
<'/h1>

<'p class="center">
This paragraph will also be center-aligned.

Note: the first character of the class name cannot use numbers! It doesn't work in Mozilla or Firefox.


As A Derivative Selector:

Like id selector, class selector also can be used as a derivative selector:
.fancy td {
color: #f60;
background: #666;
}

In this example, the larger element of the class name fancy, inside the table unit, displays orange characters in a gray background. A larger element called fancy might be a table or a div.


Elements can also be chosen based on their classes:

td.fancy {
color: #f60;
background: #666;
}

In the example above, the table cell of the class name fancy will be orange with a gray background.
<'td class="fancy">

You can assign the class fancy to any table element at any number of times. Those cells marked with fancy will be orange with a gray background. Cells that do not have a class assigned to fancy are not affected by this rule. One more thing to note is that the paragraphs of class for fancy will not be orange with a gray background, and of course, any other element that is labeled fancy will not be affected by this rule. This is due to the way we write this rule, and this effect is limited to the table cell labeled as fancy (that is, using the TD element to select the fancy class).



Element Selector

The most common CSS selector is the element selector. In other words, the elements of a document are the most basic selectors.
If you set the style of HTML, the selector will usually be a HTML element, such as P, H1, EM, a, and even HTML itself:
html {color:black;}
h1 {color:blue;}
h2 {color:silver;}

You can be a style switches from one element to another element.
If you decide to the text of the paragraph above (not H1 elements) set to gray. Just take the H1 selector to p:
html {color:black;}
p {color:gray;}
h2 {color:silver;}


Type Selectors

In the W3C standard, the element selector is also called the type selector.
The type selector matches the name of the document language element type. The type selector matches each instance of the element type in the document tree."
The following rules match all the H1 elements in the document tree:
H1 {font-family: sans-serif;}
 Therefore, we can also set the style for the elements in the XML document:


XML Document:

<'?xml version="1.0" encoding="ISO-8859-1"?>
<'?xml-stylesheet type="text/css" href="note.css"?>
<'note>
<'to>George<'/to>
<'from>John<'/from>
<'heading>Reminder<'/heading>
<'body>Don't forget the meeting!<'/body>
<'/note>
