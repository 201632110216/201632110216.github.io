Css Background





CSS allows the use of solid colors as background, and allows the use of background images to create quite complex results.
CSS's ability to do this is far beyond HTML.


Background Color

You can use the background-color property to set the background color for the element. This property accepts any legal color values. This rule sets the background of the element to gray:
p {background-color: gray;}

If you want the background color to extend from the outside of the element's text to the outside, just add some inner margins:
p {background-color: gray; padding: 20px;}

You can set the background color for all elements, including body, inline elements such as EM and a.
Background-color cannot inherit. Its default value is transparent. Transparent has the meaning of transparency. That is, if an element does not specify the background color, then the background is transparent, so that the background of the ancestral element can be visible.


Background Image

To put images into the background, you need to use the background-image property. The default value of the background-image attribute is none, which means no image is placed on the background.
If you need to set a background image, you must set a URL value for this property:
body {background-image: url(/i/eg_bg_04.gif);}
Most backgrounds are applied to body elements, but not limited to this. The following example applies a background to a paragraph without the application background of other parts of the document:
p.flower {background-image: url(/i/eg_bg_03.gif);}
You can even set the background image for inline elements. The following example sets up a background image for a link:
a.radio {background-image: url(/i/eg_bg_07.gif);}

In theory, you can even apply images to the background of replacement elements such as textareas and select, but not all user agents can handle this well.
And one more thing, background-image can't inherit it. In fact, all of the background properties are not inherited.


Background Repetition

If you want to tile the background image on the page, you can use the background-repeat property.
The attribute value repeat causes the image to be tiled horizontally and vertically, just like the usual background image. Repeat-x and repeat-y cause the image to be repeated only horizontally or vertically, while no-repeat does not allow the image to be tiled in any direction.
By default, the background image starts from the top left corner of an element. Look at the examples below:
body { background-image: url(/i/eg_bg_03.gif); background-repeat: repeat-y; }


Background location

You can use the background-position property to change the location of the image in the background.
In the following example, a background image is placed in the middle of the body element:
body { background-image:url('/i/eg_bg_03.gif'); background-repeat:no-repeat; background-position:center; }

There are many ways to provide values for the background-position attribute. First, you can use some keywords: top, bottom, left, right, and center. Usually, these keywords come in pairs, but not always. You can also use length values, such as 100px or 5cm, and finally use percentile values. There is a slight difference in the placement of background images for different types of values.


Keyword

The key to image placement is easy to understand, and its role is as the name suggests. For example, top right places the image in the upper right corner of the inner margin area of the element.
According to the specification, key position can appear in any order, as long as no more than two words - a corresponding horizontal direction, vertical direction corresponding to another.
If there is only one keyword, the other keyword is center.
So, if you want an image to appear above the middle of each paragraph, just declare as follows:
p { background-image:url('bgimg.gif'); background-repeat:no-repeat; background-position:top; }




Percentile Value

Percentile values are more complex. Suppose you want to use the percentile value to center the image in its element, which is easy:
body { background-image:url('/i/eg_bg_03.gif'); background-repeat:no-repeat; background-position:50% 50%; }
This causes the image to be properly placed and its center aligned with the center of the element. In other words, percentile values are applied to elements and images simultaneously. That is, the point (center point) described in the image 50% 50% is aligned with the point (center point) described in the element 50% 50%.
If the image is located at 0% 0%, the upper left corner will be placed in the upper left corner of the margin area of the element. If the image position is 100% 100%, the lower right corner of the image is placed on the right lower corner of the right distance.
So, if you want to put an image in the horizontal direction 2/3, vertical 1/3, you can declare that:
body { background-image:url('/i/eg_bg_03.gif'); background-repeat:no-repeat; background-position:66% 33%; }
If only a percentile value is provided, the value provided will be used as a horizontal value, and the vertical value will be assumed to be 50%. This is similar to the keyword.
The default value of background-position is 0% 0%, which is equivalent to top left in function. This explains why the background image is always tiled from the upper left corner of the inner margin of the element unless you set different position values.


Length Value

The length value explains the offset in the upper left corner of the inner margin area. The offset point is the upper left corner of the image. For example, if you set the value of 50px 100px, the upper left corner of the image will be in the upper left corner of the inner margin of the element, to the right 50 pixels, down to 100 pixels:
body { background-image:url('/i/eg_bg_03.gif'); background-repeat:no-repeat; background-position:50px 100px; }
Notice that this is different from the percentile, because the offset is only from one upper left to the other left. That is, the upper left corner of the image aligns with the specified point in the background-position declaration.


Background Association

If the document is longer, then the background image scrolls as the document scrolls down. When the document scrolls over to the image, the image disappears.
You can prevent this scroll through the background-attachment property. With this property, you can declare that the image is fixed (fixed) relative to the viewable area, so it won't be affected by scrolling:
body { background-image:url(/i/eg_bg_02.gif); background-repeat:no-repeat; background-attachment:fixed }
The default value of the background-attachment attribute is scroll, that is, by default, the background scrolls with the document.
