Css Basic Grammar






CSS Grammar

The CSS rule consists of two main parts: selectors, and one or more declarations.
selector {declaration1; declaration2; ... declarationN }

Selectors are usually HTML elements that you need to change styles.
Each declaration consists of one attribute and one value.
The property is the style attribute you want to set style attribute. Each attribute has a value. Attributes and values are separated by colons.
selector {property: value}

The following line of code is used to define the text color within the H1 element as red, and to set the font size to 14 pixels.
In this case, H1 is a selector, color and font-size are attributes, and red and 14px are values.
h1 {color:red; font-size:14px;}

Different Writing Methods And Units Of Value

In addition to the English word red, we can also use the binary color value #ff0000 sixteen:
p { color: #ff0000; }

In order to save bytes, we can use the abbreviated form of CSS:
p { color: #f00; }

We can also use RGB values in two ways:
p { color: rgb(255,0,0); }
p { color: rgb(100%,0%,0%); }
Notice that when you use the percentage of RGB, you have to write percentage symbols even when you're on duty at 0. But in other cases, you don't need to do that. For example, when the size is 0 pixels, after 0, there is no need to use the PX unit, because 0 is 0, no matter what the unit is.


Remember To Write Quotes

Tip: if the value is a few words, give the value in quotation marks:
p {font-family: "sans serif";}


Multiple Declarations:

Tip: if you want to define more than one declaration, you need to separate each statement with a semicolon. The following example shows how to define a middle paragraph of a red text. The last rule is that you don't need a semicolon, because the semicolon is a delimiter in English, not an end symbol. However, most experienced designers add semicolons at the end of each statement, and the advantage of this is that when you add or subtract statements from existing rules, you can minimize the possibility of errors. Just like this:
p {text-align:center; color:red;}

You should describe only one property on each line, so that the readability of the style definition can be enhanced, just like this:
p { text-align: center; color: black; font-family: arial; }


Space And Size

Most stylesheets contain more than one rule, and most rules contain more than one declaration. The use of multiple declarations and spaces makes the stylesheet easier to edit:
body { color: #000; background: #fff; margin: 0; padding: 0; font-family: Georgia, Palatino, serif; }

Whether or not to include spaces does not affect the work of CSS in browsers. Similarly, unlike XHTML, CSS is insensitive to case size. But there is an exception: class and ID names are sensitive to the case if they involve working with the HTML document.
