CSS Fonts





The CSS font property defines the font family, size, bold, style (such as italic) and deformation (such as small uppercase letters) of the text.


CSS Font Series

In CSS, there are two different types of font names:
Generic font series - a font system with similar appearance (e.g., Serif or Monospace)
Specific font series - specific font series (such as "Times" or "Courier")
In addition to a variety of specific font families, CSS defines 5 generic font families:
Serif font
Sans-serif font
Monospace font
Cursive font
Fantasy font


Use Universal Font Series

If you want the document to use a sans-serif font, but you don't care about which font, the following is a proper statement
body {font-family: sans-serif;}

In this way, the user agent selects a font (such as Helvetica) from the sans-serif font series and applies it to the body element. Because of inheritance, this font selection will also apply to all the elements contained in the body element unless it is covered by a more specific selector.


Specified Font Series

In addition to using the generic font family, you can also set more specific fonts through the font-family property.
The following example sets the Georgia font for all H1 elements:
h1 {font-family: Georgia;}

This rule will also have another problem, if the Georgia font is not installed on the user agent, the user agent can only use the default font to display the H1 elements.
We can solve this problem by combining specific font names and generic font families:
h1 {font-family: Georgia, serif;}

If the user does not install Georgia, but installs the Times font (a font in the serif font Series), the user agent may use Times for the H1 element. Although Times and Georgia do not match perfectly, they are at least close enough.
Therefore, we recommend that all of the font-family rules provide a generic font family. This provides a posterior path where a candidate font can be selected when the user agent fails to provide a specific font matching the rule.
If you're familiar with fonts, you can also specify a series of similar fonts for a given element. To do this, you need to arrange these fonts in precedence, and then use commas to connect them:
Example:
p {font-family: Times, TimesNR, 'New Century Schoolbook', Georgia, 'New York', serif;}
According to this list, the user agent looks up the fonts in the order in which they are listed. If all of the fonts listed are unavailable, you can simply choose an available serif font.


Quoting

As you may have noticed, single quotes are used in the examples above. Only when the font name in one or more spaces (like New York), or if the font name consists of # or $symbols, just need to add quotes in the font-family statement.
Single quotes or double quotes are acceptable. However, if you put a font-family attribute in the style property of HTML, you need to use the quotation marks that are not used by the property itself:
<'p style="font-family: Times, TimesNR, 'New Century Schoolbook', Georgia, 'New York', serif;">...


Font Style

The font-style attribute is most commonly used to specify Italic text.
This property has three values:
Normal - text normal display
Italic - Text Italic display
Oblique - text skew display

Example:
p.normal {font-style:normal;} p.italic {font-style:italic;} p.oblique {font-style:oblique;}


Font Size

Font-size property sets the size of the text.
The ability to manage text size is important in the web design domain. However, you should not make the paragraph look like a title by adjusting the size of the text, or make the title look like a paragraph.
Always use the correct HTML title, such as using <'h1> - <'h6> to mark the title, and use <'p> to mark the paragraph.
The font-size value can be absolute or relative.
Absolute value:
Set the text to the specified size
Users are not allowed to change the size of text in all browsers (which is not good for usability)
The absolute size is useful in determining the physical size of the output Relative size:
Set the size relative to the surrounding elements
Allows users to change text size in browsers
Note: if you don't specify font size, the default size of plain text (such as paragraphs) is 16 pixels (16px=1em).


Use Pixel To Set Font Size

The size of the text can be completely controlled by setting the text size of the pixel:
Example:
h1 {font-size:60px;} h2 {font-size:40px;} p {font-size:14px;}

In Firefox, Chrome, and Safari, you can reset the text size of the above example, but not in Internet Explorer.
Although you can adjust the size of the text through the browser's zoom tool, this is actually an adjustment to the entire page, not just the text.


Use em To Set Font Size

If you want to avoid adjusting text in Internet Explorer, many developers use EM units instead of pixels.
W3C recommends the use of EM size units.
1em is equal to the current font size. If the font-size of an element is 16 pixels, then 1em is equal to 16 pixels for this element. When you set the font size, the value of EM changes with the font size of the parent element.
The default text size in the browser is 16 pixels. So the default size of 1em is 16 pixels.
You can use this formula to convert pixels to em:pixels/16=em
(Note: 16 equals the default font size of the parent element, assuming that the parent element's font-size is 20px, then the formula needs to be changed to: pixels/20=em)
Example:
h1 {font-size:3.75em;} /* 60px/16=3.75em */
h2 {font-size:2.5em;} /* 40px/16=2.5em */
p {font-size:0.875em;} /* 14px/16=0.875em */
In the above example, the size of the text in EM is the same as the pixel in the previous example. However, if you use the EM unit, you can adjust the text size in all browsers.
Unfortunately, there are still problems in IE. When the text is reset, it will be larger or smaller than the normal size.


Combined Use Percentages And EM

The most effective solution in all browsers is to set the default font-size value for the body element (parent element) with a percentage:
Example:
body {font-size:100%;} h1 {font-size:3.75em;} h2 {font-size:2.5em;} p {font-size:0.875em;}
